`<zbk-pagination>` renders page navigation for a partitioned collection: a `<nav aria-label="Pagination">` wrapping a list of page items with previous/next controls, the current page marked `aria-current="page"`, and a windowed page display with non-interactive ellipses.

There is one spelling per navigation intent. With `href-template`, every item is a real link built from a `{page}` placeholder and the browser navigates — crawlers, middle-click, and open-in-new-tab all work. Without it, items are buttons and a page choice surfaces as the cancelable `zbk-page-change` event; uncanceled, the element adopts the page and announces "Page X of Y" through the shared live region.

Every control above is live: page choices fire `zbk-page-change`, and — with no listener canceling it — each element adopts the page itself.

## Usage

```html
<zbk-pagination current="3" total="12"></zbk-pagination>
```

Renders (light DOM, inspectable):

```html
<zbk-pagination current="3" total="12">
  <nav class="zbk-pagination" aria-label="Pagination">
    <ul class="zbk-pagination__list">
      <li><button type="button" class="zbk-pagination__item zbk-pagination__item--previous" aria-label="Previous page">…</button></li>
      <li><button type="button" class="zbk-pagination__item zbk-pagination__item--page" aria-label="Page 1">1</button></li>
      <li><button type="button" class="zbk-pagination__item zbk-pagination__item--page" aria-label="Page 3" aria-current="page">3</button></li>
      <li><span class="zbk-pagination__ellipsis" aria-hidden="true">…</span></li>
      <li><button type="button" class="zbk-pagination__item zbk-pagination__item--next" aria-label="Next page">…</button></li>
    </ul>
  </nav>
</zbk-pagination>
```

### Real URLs per page

```html
<zbk-pagination current="3" total="12" href-template="?page={page}"></zbk-pagination>
```

Every item becomes an `<a href="?page=N">`; there is no event to wire. A disabled previous/next control drops its `href` (a placeholder link — unreachable by Tab, announced as plain text).

### Buttons + event

```html
<zbk-pagination id="results-pages" current="1" total="20"></zbk-pagination>
<script>
  document.getElementById('results-pages').addEventListener('zbk-page-change', (event) => {
    loadPage(event.detail.page);
    // or event.preventDefault() to own the state and set `current` yourself
  });
</script>
```

## The window

The window keeps a constant width of `boundaries * 2 + siblings * 2 + 3` items, so the control never reflows while paging. An ellipsis never hides a single page — when the gap is exactly one page, the page renders instead. At the defaults (`siblings="1"`, `boundaries="1"`, `total="12"`):

| `current` | Window |
|---|---|
| 1–4 | 1 2 3 4 5 … 12 |
| 5 | 1 … 4 5 6 … 12 |
| 8 | 1 … 7 8 9 … 12 |
| 9–12 | 1 … 8 9 10 11 12 |

`siblings` sets how many neighbors flank the current page; `boundaries` sets how many pages stay pinned at each end.

## Compact

```html
<zbk-pagination compact current="3" total="12"></zbk-pagination>
```

Previous/next around a "Page 3 of 12" readout — for narrow containers where a page window cannot fit. Both navigation modes work in compact form.

## Custom control glyphs

The drawn chevrons are replaceable through the shared `icon` slot vocabulary — `data-position="start"` for previous, `data-position="end"` for next:

```html
<zbk-pagination current="2" total="5">
  <span slot="icon" data-position="start">&#8592;</span>
  <span slot="icon" data-position="end">&#8594;</span>
</zbk-pagination>
```

Glyphs are aria-hidden; the controls keep their "Previous page" / "Next page" accessible names regardless of what renders inside them.

## Accessibility

- The `<nav>` defaults its accessible name to "Pagination"; write `aria-label` on `<zbk-pagination>` when a page has more than one pagination region — it relocates to the nav.
- The current page is marked `aria-current="page"` and every item carries a "Page N" accessible name, so "7" never announces bare.
- Disabled previous/next controls use `aria-disabled` (not native `disabled`) in button mode, keeping them focusable so keyboard users don't lose their place at the ends.
- Page adoption in button mode announces "Page X of Y" through the shared live-region announcer.
- Items keep the `{a11y.min-interaction-size}` minimum tap target at every variant size, and `focus()` on the element forwards to the current page item.
